存储原理

计算机硬件最小的寻址单位是 字节 (Byte)，即 8 位二进制（$2^8 = 256$ 种组合）

  • ASCII 是为了解决英文字符的存储（只需 128 个组合）。

  • GBK/UTF-8 是为了解决**多语言（如中文）**上万个字符的存储。

ASCII码

ASCII编码参考表
American Standard Code for Information Interchange

原理

  • 宽度： 固定 1 字节。

  • 范围： 0 - 127 (00000000 - 01111111)。

  • 原理： 最高位（第 8 位）永远是 0。它只能存 26 个英文字母、数字和标点。

  • 局限： 无法表示中文，因为中文字符远超 256 个。

GBK

原理

  • 宽度： 变长编码（1 或 2 字节）。

  • 原理： 为了兼容 ASCII，GBK 做了“高位判定”：

    1. 如果读取到的字节最高位是 0，则认为它是 ASCII，占 1 字节。

    2. 如果最高位是 1，则认为它是汉字的“高字节”，必须和紧随其后的下一个字节拼接起来，组成一个 16 位的数字来表示一个汉字。

存储原理

GBK 并不直接使用 Unicode 码点，它有自己的映射表。在 GBK 中，“中”的编码是 D6 D0（十六进制）。

  1. 二进制化： D6 是 11010110，D0 是 11010000。

  2. 存储： 磁盘上连续写入 11010110 11010000。

  3. 读取： CPU 读取第一个字节，发现最高位是 1，知道“这是一个汉字的开头”，于是连读两个字节，查 GBK 表，显示出“中”。

Docker概念_命名空间
Docker 并不是一种全新的虚拟化技术，它本质上是一个利用 Linux 内核特性实现的隔离进程。 Docker 之所以能让容器看起来像一个独立的操作系统，全靠 Namespaces（资源隔离） 和 Cgroups（资源限制） 这两项核心技术。 基本定义命名空间（Namespaces） 是 Linux 内核的一个特性，用于对内核资源进行物理隔离。 通俗解释：想象你在一栋办公楼里。命名空间就像是给每个团队分配了一个独立的办公室。虽然大家都在同一栋楼（同一个 Linux 内核）里，但 A 团队看不见 B 团队的桌面，也听不见他们的电话。 运维意义：它保证了容器 A 里的进程编号、网络配置、挂载点等，不会与容器 B 或宿主机发生冲突。 七大命名空间PID Namespace-进程隔离NET Namespace-网络隔离MNT Namdespace-挂载点隔离UTS Namdespace-主机名隔离IPC Namdespace-进程间通信隔离USER Namespace-用户隔离CGROUP Namdespace
