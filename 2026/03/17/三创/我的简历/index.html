项目经历

日志收集与流量预警系统

  • 项目时间：2025/12-2025/01
  • 项目环境：CentOS Stream 10 ， Kafka 3.6.1 ， Mariadb ，celery
  • 项目描述：本项目旨在构建基于 Kafka 的日志收集与流量预警平台，负责从 Web 集群的 Nginx 反向代理服务器中采集日志并发送至 Kafka，通过编写消费者程序对 Kafka 中的日志进行处理，提取流量信息。基于 Celery 实现流量异常的自动化预警机制，并结合 ELK 技术栈对流量数据进行可视化展示，提升系统监控效率与响应速度。
  • 项目流程：
    • 搭建并优化项目运行环境，设计系统整体架构
    • 部署 Kafka 集群，确保高可用与数据吞吐能力
    • 协助搭建前端 Web 集群，保障日志采集稳定性
    • 编写 Kafka 消费者程序，实现日志解析与流量数据提取
    • 集成 Celery 任务队列，实现流量异常的实时监控与告警
    • 使用 ELK 技术栈完成流量数据的可视化展示，便于运维人员快速掌握系统状态
  • 项目成果：
    • 加深了对kafka，nginx，filebeat，celery等技术的运用与理解，提升故障排查能力与对日志收集系统整体框架的认知
    • 积累了自动化告警的经验

高速公路车辆识别与分析系统

  • 项目时间：2025/3-2025/5
  • 项目环境：Python ，ultralytics (YOLOv8) ，gradio ，OpenAI SDK ，MySQL ，pyecharts ， pandas ，swanlab
  • 项目描述：针对高速公路复杂交通场景，开发了一套集“实时感知、结构化存储、自然语言分析”于一体的智能交通系统。系统实现了对 12 种细分车型的分类检测，并允许用户通过自然语言（Text-to-SQL）调取数据库进行交通数据可视化分析。
  • 核心职责：
    • 垂直领域模型训练：基于 YOLOv8 架构，针对 12 种细分车型（如超长卡车、长/短公交等）进行模型训练。通过优化 imgsz=512 等超参数，实现了复杂背景下的高精度目标检测。
    • 端到端交互系统开发：利用 Gradio 构建了可视化 Web 后台，集成置信度与 IOU 动态调节功能，实现了 ms 级推理延迟统计及检测结果的实时标注渲染。
    • 异构数据结构化存储：设计并实现了自动化的数据持久化逻辑，利用 pymysql 将检测到的车辆类别、置信度及坐标信息（xmin, ymin, xmax, ymax）实时存入 MySQL 数据库，为后续分析累积结构化交通大数据。
    • 创新性 Text-to-SQL 模块：接入 DeepSeek-V3 大语言模型，设计了针对交通业务的提示词工程（Prompt Engineering），实现了从自然语言需求到 SQL 语句的自动转换（如“查询今日上午轿车数量比例”）。
    • 自动化 BI 报表生成：集成 pyecharts 动态生成热点车型分布图、饼图等可视化图表，并驱动 AI 自动撰写专业的数据分析报告，大幅提升了非技术人员的数据决策效率。

混合云架构的web高可用自动化集群部署

  • 项目时间：2025/12-2026/3
  • 项目地址：https://yh1140.github.io/categories/web综合集群
  • 技术栈：Linux / FRP / Ansible / Docker / k8s / Nginx / keepalived / MySQL 8.4 LTS / NFS / Prometheus + Grafana + Node_Exporter / Kafka / ELK /
  • 项目描述：该项目模拟真实生产环境，构建了一个“公有云管理私有云”的混合云架构。利用火山云服务器作为全局管理节点（Cloud-Hub），通过 FRP 安全隧道内网穿透，管理包含防火墙、负载均衡、Web 服务器、数据库及 NAS 存储在内的 7 台以上主机。项目实现了从基础设施自动化配置到业务高可用上线全流程，重点解决了复杂网络环境下的集中式自动化管理与数据强一致性问题。
  • 核心内容：

    • 混合云安全链路构建：利用 FRP 建立云对本地隧道，通过 云主机 配合 SSH ProxyCommand 实现对内网 Host-Only 模式主机的透明化管理，确保管理链路的安全性与隔离性。

    • 基础设施自动化 (IaC)：基于 Ansible 编写全套环境初始化 Playbook，实现跨节点的一键同步初始化配置（SElinux，镜像源，基础工具，时间同步等），大幅提高集群初始部署效率。

    • 网络中枢与安全控制：在本地独立部署 Firewall 节点，利用双网卡结构（NAT+Host-Only）承担 Router 与网关角色，通过定制化脚本实现 SNAT/DNAT 转发，精细化控制进出流量。

    • 高可用存储与数据库设计：部署 NAS (NFS v4.2) 解决 Web 集群间静态资源同步问题；配置 MySQL 8.4 MGR (组复制) 模式，实现了数据强一致性和故障秒级自动切换，解决了数据库单点故障风险。

    • 监控与自愈能力：整合 Prometheus 生态实现全链路监控，利用 Kafka+ELK 实现日志收集与可视化。利用 K3s 配合 Metrics-Server 实现基于 CPU 利用率的 HPA 自动扩容，提升了系统在高峰期的承载能力。

  • 项目心得：
    • 自动化与一致性：从学习阶段的单节点部署到项目的脚本批量配置与部署以及数据统一挂载，让我深刻理解了脚本化以及一致性是大规模集群中的关键。
    • 云原生思维：容器化部署以及探针，扩容，调度等措施，极大提升了集群健壮与自愈性。
    • 实战验证：完整跑通了“流量采集-消息缓冲-异步告警-可视化”的工业级日志链路，沉淀了 2 万字的排障与配置文档。
    • 稳定性表现：在模拟 1000 并发压测下，通过 Kafka 成功缓冲并处理了 10 万次请求产生的日志，系统未出现宕机或日志积压。
    • 规范化输出：掌握了生产环境下配置文件版本管理、系统服务（systemd）封装及基础的安全加固习惯。
