Docker概念_什么是容器

发表于2026-03-17|更新于2026-04-08|Docker容器
|浏览量:

容器概念

本质

宿主机上一组隔离的进程，共享主机内核，轻量、隔离且高效，不像虚拟机需要完整的操作系统

容器特性

共享-依靠 共享内核 & Cgroups

这是容器与虚拟机最大的区别

内核共享

所有容器直接运行在宿主机的内核上
共享主机的硬件&全局系统资源：

  • 系统时间
  • 内核参数
  • 内存段
  • 信号量
  • 消息队列
  • 目录/文件（卷）
  • 网络资源
  • 容器间link
    优点：极快，秒级启动，几乎零损耗。
    缺点：如果一个进程触发了内核漏洞（Kernel Panic），整个宿主机（包括所有容器）都会崩掉。

资源限制

既然共享资源，就得“分家产”。Cgroups（Control Groups） 负责限制进程能用多少 CPU、内存和磁盘 I/O

隔离-依靠 Namespace

在 K8s 中，主要的隔离维度包括：

PID Namespace

容器只能看到自己的进程，看不到宿主机或其他容器的进程。

Net Namespace

每个 Pod 拥有独立的 IP 和端口空间。

Mount Namespace

容器只能看到自己的根文件系统（Rootfs）。

UTS Namespace

独立的 hostname，区分环境。

容器架构

物理架构

逻辑架构 - 经典的 C/S 模型

Docker 在逻辑上是一个典型的客户端-服务器应用。

  • Docker Client (客户端)：这是你操作的入口（如 docker run）。它并不执行具体的容器逻辑，而是通过 REST API 发送请求。

  • Docker Daemon (守护进程 - dockerd)：这是宿主机上的后台进程。它监听来自 Client 的 API 请求，并管理镜像、容器、网络和磁盘卷。

  • Docker Registry (镜像仓库)：存放镜像的“图书馆”。在你的环境下，它是“渡渡鸟”镜像站。

分层架构

容器家族

核心容器引擎Container Engines

Docker Engine (Moby)

  • 地位： 行业标准，最流行。
  • 特点： 易用性极高，拥有庞大的社区和镜像仓库 (Docker Hub)。虽然 Docker 公司改变了桌面版收费策略，但核心的 Docker Engine (Moby 项目) 依然是开源的。
  • 现状： 依然是开发者本地开发的首选。

containerd

  • 地位： 云原生计算基金会 (CNCF) 毕业项目，Kubernetes 的默认运行时。
  • 特点： 从 Docker 中剥离出来的核心组件，比 Docker 更精简、稳定，专为大规模生产环境设计。
  • 现状： 大多数生产环境的 K8s 集群底层其实都在跑 containerd。

CRI-O

  • 地位： Red Hat 主导，专为 Kubernetes 打造。
  • 特点： 轻量级，只实现了 K8s 的 CRI (容器运行时接口)，不依赖 Docker 的代码库。
  • 现状： 在 Red Hat OpenShift 等环境中广泛使用。

Podman

  • 地位： Docker 的主要替代品之一 (Red Hat 主推)。
  • 特点： 无守护进程 (Daemonless) 架构，更安全。命令行与 Docker 完全兼容 (alias docker=podman)。
  • 优势： 可以在没有 root 权限的情况下运行容器 (Rootless)，这对安全性要求高的环境很有吸引力。

LXC / LXD

  • 地位： 老牌容器技术。
  • 特点： 侧重于系统级容器 (System Containers)，即模拟一个完整的 Linux 虚拟机环境，而不是像 Docker 那样侧重于单个应用。

容器编排平台Orchestration

Kubernetes (K8s)

  • 地位： 绝对的霸主，容器编排的事实标准。
  • 特点： 功能极其强大，几乎能应对所有复杂场景，但学习曲线陡峭。
  • 云厂商支持： AWS (EKS), Azure (AKS), GCP (GKE), 阿里云 (ACK) 等均基于 K8s。

Docker Swarm

  • 地位： Docker 原生的编排工具。
  • 特点： 极其简单，与 Docker 集成度高，但功能远不如 K8s，目前市场份额已大幅萎缩，仅用于简单场景。

HashiCorp Nomad

  • 地位： 通用的调度器。
  • 特点： 不仅仅能调度容器，还能调度虚拟机、应用程序。架构比 K8s 简单，部署维护成本低。

轻量级/下一代容器

Kata Containers / gVisor

  • 特点： 强隔离容器。结合了虚拟机的安全性（拥有独立内核）和容器的轻量级。适合多租户环境，防止容器逃逸。

MicroVMs (Firecracker / AWS Lambda)

  • 特点： 亚马逊开源的技术，用于 AWS Lambda 和 Fargate。它是基于 KVM 的微型虚拟机，启动速度像容器一样快，但安全性是硬件级的。

Wasm (WebAssembly) 容器

  • 新星： Spin, Wasmtime, Docker+Wasm。
  • 特点： 比 Linux 容器更轻量、启动更快、可移植性更强（不仅限于 Linux）。目前主要用于运行微服务和无服务器函数，正在逐渐挑战传统 OCI 容器的地位。

桌面级

Docker Desktop

  • 现状： 功能最全，但对大型企业收费。

Rancher Desktop

  • 现状： 开源免费，内置了 k3s 和 containerd/Podman，是 Docker Desktop 的主要免费替代品。

Colima
