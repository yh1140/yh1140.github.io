###基本概念###

#route 路由

#router 路由器-将一个网段的数据转发到另一个网段

网络世界的立交桥分流器和邮局调度员。它的核心任务只有两个：

维护地图（路由表）

转发数据包（寻址）

##核心功能

路由器工作在 OSI 模型的第三层（网络层），其功能可以概括为以下三点

  • 路径选择（寻址）：当一个 IP 包到达时，路由器会查看目的 IP 地址，查阅自己的“地图”（路由表），决定该把它发往哪个方向。
  • 网络互连：路由器是连接不同网络的“桥梁”。它可以把家庭局域网（LAN）和互联网（WAN）连在一起，也可以连接两个完全不同的物理介质（如光纤和以太网）。
  • 隔离广播风暴：与交换机不同，路由器不会转发广播包。这意味着它能把网络故障限制在局部，不至于让整个公司的网络瘫痪。

##硬件构造

#本质是一个主机

  • 处理器 (CPU)：负责运行路由协议、计算最优路径。
  • 内存 (RAM/ROM/Flash)：存放操作系统（如 Cisco IOS）、运行中的路由表和缓存数据。
  • 接口 (Interfaces)：
  • LAN 口：连接你的电脑、交换机。
  • WAN 口：连接宽带拨号猫（Modem）。
  • Console 口：给工程师调试用的串口。

##路由表

路由表里有什么

如果你进入路由器的后台输入指令（如 show ip route），你会看到一排类似代码的条目。一条标准的路由表项通常包含以下核心要素：

  • 目标网络/掩码 (Destination/Mask)：你要去的小区地址（例如 192.168.1.0/24）。
  • 下一跳 (Next Hop)：要去那个小区，中间经过的下一个“车站” IP。
  • 出接口 (Interface)：数据包应该从路由器的哪个物理端口“踢”出去（如 GigabitEthernet 0/1）。
  • 管理距离 (Administrative Distance, AD)：可信度。如果两个来源都告诉路由器怎么走，它会选 AD 值小的（越小越靠谱）。
  • 度量值 (Metric)：开销。如果路一样靠谱，它会选路费最便宜（路径最短、网速最快）的一条。
路由表的路由类型

① 直连路由 (Connected Routes)

  • 来源：只要你给路由器的物理接口配上 IP 并接通电源，它就会自动生成这条路由。
  • 特点：可信度最高（AD=0），因为它是路由器“亲眼看到”的邻居。
  • 场景：连接公司内部的不同部门。

② 静态路由 (Static Routes)

  • 来源：由网络管理员手工一条条敲进去的。
  • 特点：非常稳定且不占带宽。但它很“死板”，如果中间路断了，它不会自动绕路。
  • 默认路由 (Default Route)：这是一种特殊的静态路由，写法通常是 0.0.0.0/0。它的意思是：“如果你在表里找不到目的地，就把包全都扔给我。” 它是连接互联网的关键。

③ 动态路由 (Dynamic Routes)

  • 来源：路由器之间通过“聊天”（运行协议）互相交换地图。
  • 特点：能自动适应网络变化。如果一条路断了，它们会自动商量出一条新路。
  • 常见协议：
    • OSPF：最流行的企业内网协议。它像“高德地图”，计算整张网的拓扑图，走带宽最快的路。
    • BGP：互联网的 “总管”。它负责不同国家、不同运营商（如电信、联通）之间的超大规模路由。
#优先级

直连路由->静态路由->动态路由->默认路由

###工作原理###

当一个 IP 包进入路由器，它会经历以下过程

  • 解封装：拆掉二层“帧”的外壳，露出里面的 IP 包。
  • 查表（最长匹配原则）：
  • 路由器查看目的 IP（例如 159.64.25.100）。
  • 在路由表中寻找最匹配的网段。
  • 如果找到，就转发；如果没找到，就扔给“默认网关”；如果连默认网关都没有，就丢弃并报错。
  • 重新封装：根据下一跳的方向，给 IP 包换上新的“帧”外壳（新的目标 MAC 地址）。
  • 转发：从相应的物理接口发出去。

## ##动态路由协议-工作算法分类 ### 距离矢量类 (Distance Vector)
概念

#原理： 路由器并不掌握全网的拓扑结构，它只知道两个参数：方向（Vector）和距离（Distance）。

  • 传闻式路由：路由器定期将自己的整个路由表发送给直接相连的邻居。
  • 邻居关系：它盲目相信邻居给出的信息。如果邻居说“去 A 网段走我这儿只要 2 跳”，它就会记录下来。

#优缺点

  • 优点：配置简单，对 CPU 和内存的消耗极低。
  • 缺点：
    • 收敛慢：信息像“传话游戏”一样一站站传递，全网达成一致需要很长时间。
    • 容易产生环路：由于不掌握全局，容易出现 A 以为去目标走 B，B 以为走 A 的死循环。
    • 跳数限制：为了防止无限循环，通常有最大跳数限制（如 RIP 的 15 跳）。
代表


### 链路状态类 (Link State)
概念

#原理： 每个路由器都会产生链路状态通告 (LSA)，说明自己连接了哪些接口、带宽是多少

  • 同步地图：路由器将这些 LSA 广播给全网所有路由器，最终每个路由器手里都有一张一模一样的全网地图（LSDB，链路状态数据库）。
  • 自主计算：每台路由器以自己为根节点，利用 Dijkstra 算法（SPF，最短路径优先算法） 计算出通往每个目的地的最短路径。

#优缺点

  • 优点：
    • 无环路：因为每个路由器都看得到全图，所以不会被错误的传闻误导。
    • 收敛极快：一旦网络变动，触发式更新迅速扩散，全网秒级感知。
    • 选路精准：通常基于“开销（Cost，与带宽有关）”计算，而不是死板的跳数。
  • 缺点：算法复杂，对路由器的 CPU 和内存要求较高（但在现代设备上这已不是问题）。

## ##动态路由工作流程 + **发现邻居**：路由器启动后，发一个“Hello”包：嘿，旁边有人吗？ + **建立关系**：邻居回应，双方交换基本的身份信息。 + **同步地图**：双方交换自己知道的所有链路状态信息（LSA），确保大家的“全网地图”一模一样。 + **计算路径**：每台路由器在脑子里跑一遍算法，选出最优路径存入路由表。 + **实时维护**：如果有线路断了，第一时间通知全网，大家重新算路。

##概述

RIP（Routing Information Protocol，路由信息协议）是网络中最经典、最简单的动态路由协议之一。

#主要用于小型网络，其核心逻辑可以概括为一句话：“传声筒”式的距离矢量算法

**# 核心衡量标准：跳数 (Hop Count) **

  • 每经过一台路由器，跳数就加 1。
  • 最大跳数为 15。如果跳数达到 16，RIP 就认为目标网络不可达。
  • 这就限制了它只能用于小型网络

##工作原理

1、RIP路由协议每隔30s向邻居发送整个路由表信息

2、RIP路由协议以跳数作为度量值，根据跳数的多少来选择最佳路由

3、最大跳数为15跳，16跳被标记为不可达

4、经过一系列路由更新，网络中的每个路由器都具有一张完整的路由表的过程，称为收敛

##三种计数器

1、周期更新计时器（update time）

即路由器每隔30s从每个启动RIP协议的接口发送路由更新信息。

2、路由老化计时器（age time）

若一条路由在180s内没有收到更新，这条路由的跳数将记为16（即不可达状态）。

3、垃圾收集计时器（garbage collection time）

若该路由在被标记为16跳后，120s内还没有收到更新，则将这条路由从路由表中删除。

##路由环路问题

环路产生

假设有三台路由器：A --- B --- C。

  1. 正常状态： C 告诉 B 它连接着网络 192.168.1.0（跳数为 0），B 记录下该路径（跳数为 1），并传给 A（跳数为 2）。
  2. 链路故障： C 发现 192.168.1.0 断开了，它立即删除了这条路由。
  3. 信息差错： 在 C 还没来得及把坏消息告诉 B 之前，B 的定期更新时间到了。B 告诉 C：“嘿，我有去 192.168.1.0 的路，跳数是 1。”
  4. 死循环开始： C 以为 B 有另一条路能通，于是更新路由表，记录跳数为 2。接着 A 又从 C 那里学到，跳数变成 3。
  5. 计数到无穷大： 这个错误信息在三者之间不断传递，跳数不断增加（1, 2, 3... 15），直到达到 16（不可达） 才会停止。在这个过程中，发往该网络的数据包会一直在 A、B、C 之间转圈。
解决方法

为了防止这种“逻辑混乱”，RIP 设计了一套防御机制：

① 最大跳数 (Maximum Hop Count)

  • 逻辑： 规定跳数最大为 15，16 即为死亡。
  • 作用： 确保环路不会永远持续下去，最终会因为“数到 16”而终止。但它的缺点是收敛太慢，网络会瘫痪一段时间。

② 水平分割 (Split Horizon)

  • 逻辑： “从哪个接口学到的路由，就不再从这个接口发回去。”
  • 作用： 在上面的例子中，B 从 C 那里学到了网络信息，就不会再把这条信息传回给 C。这是防止环路最基本、最有效的方法。

③ 毒性逆转 (Poison Reverse)

  • 逻辑： 当 A 发现某个网络断了，它不只是停止通告，而是直接告诉邻居 B：“那个网络现在的跳数是 16！”
  • 作用： 强行让邻居把坏消息“吃下去”，彻底切断邻居对该路径的幻想。

④ 触发更新 (Triggered Update)

  • 逻辑： 不再死板地等待 30 秒周期。只要网络拓扑一变，路由器立即向外发送更新报文。
  • 作用： 抢在错误信息传播之前，把正确的故障信息发出去，大大缩短了产生环路的时间窗口。

##自洽系统

#自治系统（Autonomous System ）AS：一些路由器工作在同一个范围内，路由器之间可以相互通信且运行相同的路由协议，遵从相同的选路规则和依据，那么我们就称这些路由器工作在同一个自治系统内部。

所谓的外部网关协议，其实就是用来连接不同的AS的

#OSPF属于内部网关路由协议，所谓内部网关路由协议是指工作在企业内网中的协议，更准确的说是用于单一自治系统内的协议

##OSPF基本概念

①OSPF没有路由器个数的限制，被称之为叫做无限广大的路由协议

②OSPF为了防止单点故障引发的问题影响整个网络，因此其引入了区域【Area】的概念

③OSPF规定，每个区域中最多可以包含255台路由器，但是OSPF没有区域个数的限制

④在OSPF中，区域0被称之为骨干区域，除区域0外，其它区域被称之为非骨干区域

⑤非骨干区域必须能够通过一个ABR【Area BorderRouter】区域边界路由器】与骨干区域直接相连，若没有ABR负责联通骨干区域与非骨干区域，则该非骨干区域无法与骨干区域通讯

⑥OSPF可以没有非骨干区域，所有路由器均工作在同一个区域0中（OSPF的单区域网络）

⑦若一台路由器的所有接口均工作在同一个区域中，则该路由器被称之为内部路由器；若该区域为区域0，则该路由器还可以被称为骨干路由器

⑧若一台路由器的不同接口属于不同的区域，则该路由器被称之为区域边界路由器（ABR）ABR的作用是用来在多个区域之间相互传递彼此的汇总路由信息的

⑨OSPF区域的划分是在接口上的

使用场景&特点

#使用场景

网络规模：网络中的路由器个数在10台以上，中等或大规模的网络部署

网络拓扑结构：网络拓扑结构为网网状，并且任意两台路由器之间都有互通的功能。

其他特殊需求：要求路由变化时能够快速收敛，要求路由协议自身的网络开销尽量降低

对路由器自身的要求：对路由器自身的CPU和内存性能要求较高，配置低的路由器不建议配置OSPF

## ##OSPF工作原理 #OSPF协议是标准的链路状态路由协议

第一步： 建立邻居关系 (Hello 阶段)

路由器启动后，首先要寻找周围有哪些运行 OSPF 的邻居。

  • 发送 Hello 报文： 路由器通过接口向外发送 Hello 包，包含自己的 Router ID (RID)。
  • 参数匹配： 邻居之间必须在某些关键参数（如 Area ID、认证信息、时间间隔）上达成一致。
  • 形成邻居表： 一旦握手成功，双方进入 2-Way 状态，确认彼此的存在。

第二步：同步链路状态数据库 (LSDB 阶段)

#这是 OSPF 最核心的步骤。目标是让同一区域（Area）内的所有路由器拥有一模一样的地图。

  • LSA (链路状态通告)： 每台路由器都会产生 LSA，描述自己的接口 IP、掩码、开销以及连接了哪些邻居。
  • LSU (链路状态更新)： 路由器将 LSA 打包发送给邻居。
  • 泛洪 (Flooding)： 收到新 LSA 的路由器会将其转发给除了来源接口以外的所有其他接口，确保全区域同步。
  • LSDB： 最终，所有 LSA 存储在链路状态数据库中。此时，每台路由器都看清了整个区域的拓扑结构。

第三步：计算最优路径 (SPF 计算)

#有了完整的地图（LSDB）后，路由器不再听信邻居的结论，而是自己动脑筋。

  • Dijkstra 算法： 路由器以自己为根节点，运行 SPF（最短路径优先）算法。
  • 计算 Cost： 它是根据带宽计算开销，公式通常为 $Cost = \frac{10^8}{Bandwidth}$。
  • 生成路由表： 算法会计算出到达每个网段的路径中，总 Cost 值最小的那一条，并将其放入路由表。

邻接关系建立流程&状态

#在整个工作原理中，路由器会经历多个状态的转变：

  1. Down / Attempt: 初始状态。
  2. Init: 收到邻居的 Hello 包，但包里还没写我的名字。
  3. 2-Way: 双方都看到了对方，建立了邻居关系。
  4. ExStart / Exchange: 决定谁先发送“地图目录”（DBD 报文）。
  5. Loading: 发现自己缺少的地图碎片，向邻居请求详细的 LSA。
  6. Full: 数据库完全同步，这是 OSPF 正常工作的最终状态。
邻接关系建立流程

#这其中一共用到了7种状态和5种报文

如下图所示。RTA和RTB都处于down状态，down状态表明两个人谁也不认识谁，可以理解为OSPF还没配好。

假设RTA的OSPF先配好了，于是RTA先给B发送了hello报文，相当于A给B打了个招呼说你好我是路由器A，在A给B发送的hello报文中包含了RTA自己的名字，RTB在收到hello报文之后将自己的状态变成了init状态。

然后B也会给A返回一个hello报文，里边包含了B的名字。A收到B的hello报文之后，也将自己的状态变为init状态。然后A在给B发送一个hello报文，里边包含了B的名字，相当于说你好我知道了你的名字是B。

此时B看到A的hello报文之中包含自己的名字，B会认为A肯定已经知道它的存在了。然后B将自己的状态转变为2-way双向状态。同时B也会给A发送第二个hello报文，里边包含了A的名字。A在收到这个报文之后也将自己的状态转为2-way双向状态。

然后A会向B发送DBD数据库描述报文，该报文中是空的，啥也没有，B在收到这个空的报文之后将自己的状态转为exstart准启动状态。同时B也给A发送一个空的DBD数据库描述报文，A收到之后也将自己的状态转为exstart准启动状态。A和B发送空的DBD数据库描述报文的目的是为了协商主从关系，谁的路由器ID数值更大，谁就是主路由器。确保主从关系的目的是为了确认下一个包谁先给谁发，对于OSPF来讲，永远都是主先给从发。

假设A是主路由器，A会先给B发送第二个DBD报文，这个报文不在是空的，里边包含RTA直连网段的汇总信息，B在收到这个信息之后会将自己的状态从准启动状态转变为准交换状态，从这个状态开始B知道了它要准备开始和A交换路由信息了。同时B也会给A发送第二个DBD报文，里边包含RTB直连网段的汇总信息，A在收到这个信息之后会将自己的状态从准启动状态转变为准交换状态。到这为止，相对于两个人互换了目录。

然后A会给B发送链路状态请求报文LSR，B在收到这个报文之后会给A回复链路状态更新报文LSU，A每收到一个更新，就会给B回复一个LSACK确认报文，确认我收到了你的更新。A在学习B的所有路由的时候，A的状态处在loading加载状态。B也会给A发送链路状态请求报文LSR，A在收到这个报文之后会给B回复链路状态更新报文LSU，同样B每收到一个更新，就会给A回复一个LSACK确认报文，确认我收到了你的更新。B在学习A的所有路由的时候，B的状态也处在loading加载状态。

最终当A学完了B的路由，B也学完了A的路由，A和B都进入full全毗邻状态，全毗邻状态表明两个路由器已经彻底的完成了邻接关系的建立。

OSPF邻接关系需要满足的条件

那么是不是任意两个OSPF路由器都能建立邻接关系呢？当然不是，它需要满足一定的条件才可以。

1、路由器必须在同一个区域中，OSPF不允许跨区域建立邻接关系

2、路由器相连的两个端口上必须配置相同的密钥，或者不配置

3、OSPF协议需要两个邻居路由器的hello和dead时间间隔相同，否则就不能成为邻居路由器

### RID #在OSPF当中，要求每个路由器必须得有个名，这个名就是 router ID

# 在 OSPF 的世界里，RID (Router ID) 就是路由器的“身份证号”。它是一个 32 位的无符号整数，格式通常写成类似 IP 地址的点分十进制（例如 <font style="color:rgb(51, 51, 51);">1.1.1.1</font>）

# 在 OSPF 区域内，每台路由器必须有一个全网唯一的 RID，否则邻居关系无法正常建立，拓扑计算也会出错

router自动选取RID规则

1、若路由器上配置了环回（loopback）接口，则环回接口的地址就是该路由器的RID

环回接口是一个虚拟接口，一般有2个用途

一个是可以用来给路由器命名，在为路由器命名时，一般配置为32位的子网掩码。

另一个作用是用来模拟一个网段，或者模拟一个具体的主机地址，用来测试网络的连通性。

2、路由器上可以配置多个环回接口，如果路由器上有多个环回接口时，那么谁的IP数值最高，谁就是RID

3、如果没有环回接口，就在所有物理端口中选取一个数值最高的IP地址。

### 工作表 #运行OSPF路由协议的路由器内部要维护三张表。

①邻接关系表

列出每台路由器全部已经建立邻接关系的邻居路由器。所以邻接关系表里保存的是邻居路由器的RID

②链路状态数据库

列出网络中其他路由器的信息，由此显示全网的网络拓扑结构

③路由表

列出通过SPF算法计算出的到达每个相连网络的最佳路径


#邻接接关系表中保存的是邻居的路由器ID，链路状态数据库中保存的是最全的路径，路由表中保存的是最佳的路径。 这三张表的关系不是独立的，他们之间有非常紧密的关系。有了第一个表才能得到第二个表，有了第二个表才能得到第三个表。

##DR&BDR

#把D选为班长，也就是指定路由器DR，让其他的路由器只和班长建立邻接关系，而无需他们彼此之间建立邻接关系，从而大大降低了要建立邻接关系的数量

#如果指定路由器挂了，就需要选出一个班副来，即备份指定路由器BDR。。其他的路由器只和班长和班副即DR和BDR建立邻接关系。

#那么DR和BDR是怎么选择出来的呢？所有运行OSPF的路由器都有一个OSPF优先级，优先级最高的路由器会被选举为DR，如果OSPF优先级相同，则具有最高RID的路由器会被选为DR。优先级的取值范围为0-255，且默认值都为1。数值第二高的就被选为BDR。如果DR和BDR都挂了，那么会在剩下的路由器中重新选举

##OSPF度量方法

RIP根据跳数选择最佳路径，OSPF根据什么选择最佳路径呢？

OSPF会选择带宽最大的路径作为最佳链路，根据值是cost值，cost=10的8次方/带宽，所以带宽越大cost值越小，cost值最小的会被选为最佳路径。一条链路上的所有cost值之和就是该链路的总cost值。如果两台链路带宽cost值完全一样，那么这两条路都会传输数据，因为OSPF支持等价负载均衡。
