④配置-多端操控firewall主机

利用 VMware 的 NAT 映射功能，把物理机的某个端口“透传”给 Firewall 虚拟机，实现云服务器能够访问本地firewall主机

端口转发-让物理机连接firewall主机

  • 打开 VMware Workstation -> 编辑 -> 虚拟网络编辑器。
  • 选择 NAT 模式 的那块网卡（通常是 VMnet8），点击 NAT 设置。
  • 在“端口转发”栏点击 添加：
    • 主机端口：2233（这是你物理机开启的端口）
    • 类型：TCP
    • 虚拟机 IP 地址：192.168.10.125
    • 虚拟机端口：22
  • 确定并保存。
  • 在你物理机（Windows）上打开 CMD，输入 ssh root@127.0.0.1 -p 2233。如果能连上 Firewall，说明映射成功。

⑤验证

物理链路层

确保两块网卡不仅有 IP，而且状态是 UP，且没有配置冲突。

  • 命令：nmcli device status

  • 指标：ens224 (WAN) 和 ens160 (LAN) 必须都处于 connected 状态。

  • 命令：ip a

  • 指标：

    • ens224: 应该只有 192.168.10.125/24。

    • ens160: 应该只有 192.168.31.2/24。

网络层

对于路由器来说，路由表就是它的灵魂。我们要确保数据包知道“去哪儿”。

  • 命令：ip route show

  • 指标：

    • 必须有一条：default via 192.168.10.2 dev ens224（这是通往外网的唯一出口）。

    • 不能有第二条 default 开头的路由（特别是不能指向 ens160）。

    • 直连路由：应该有 192.168.10.0/24 指向 ens224，192.168.31.0/24 指向 ens160。

传输层&应用层

ping 只能证明网络通了，curl 才能证明 Web 业务流量能正常进出。

  • 命令：curl -I https://www.baidu.com

  • 指标：返回 HTTP/1.1 200 OK。这证明了：

    1. DNS 解析正常（将域名转成了 IP）。

    2. TCP 三次握手成功。

    3. HTTPS/SSL 握手成功（如果时间不对，SSL 握手会失败，说明你的服务器时间需要同步）。

路由转发&NAT策略检查

这是为了验证这台机器是否具备“带人上网”的能力。

  • 内核转发状态：

    • 命令：sysctl net.ipv4.ip_forward

    • 指标：必须为 1。

  • NAT 规则统计：

    • 命令：iptables -t nat -L POSTROUTING -n -v

    • 指标：看 pkts（包数量）那一列。如果你现在去内网机器执行一次 ping，这个数字应该会跳动。

DNS稳定性

  • 命令：cat /etc/resolv.conf

  • 指标：包含 nameserver 114.114.114.114。

  • 专家提示：在 Rocky 9 中，如果你通过 nmcli 配置了 DNS，这个文件会被自动管理。如果手动改它，重启后会失效。
