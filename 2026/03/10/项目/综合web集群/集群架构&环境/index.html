组件

监控系统 Prometheus 3.0 时序数据库监控 负责全集群数据抓取
可视化 Grafana 11.x 展示监控仪表盘 导入 8919/1860 中文模板
共享存储 NAS (NFS) 网络文件系统 核心网页源码共享中心

前置条件

硬件资源

  • 内存：宿主机（笔记本）建议 16GB 及以上。若所有 7 台虚拟机同时开启，建议每台分配 1GB-2GB 内存。
  • BIOS 设置：确保你的电脑 BIOS 中已开启 VT-x (Virtualization Technology) 虚拟化支持。

宿主机网络环境配置 (VMware)

  • VMnet1 (Host-Only)：进入 VMware 的“虚拟网络编辑器”，将 VMnet1 的子网 IP 设置为 192.168.10.0。
  • 静态 IP 预留：关闭 VMnet1 的 DHCP 功能，我们将手动为每台机器分配 .191 之后的静态地址，以防 IP 漂移 。

云服务器安全组配置

登录火山云控制台，在安全组中开放以下端口：

  • SSH (22)：用于你连接云服务器。
  • Grafana (3000) & Prometheus (9090)：用于外部查看监控图表 。
  • ICMP：确保能与本地 Firewall 进行 Ping 测试。

本地防火墙基础镜像准备

  • 建议先创建一个干净的 Rocky 9 最小化安装镜像。
  • 安装基础工具包：yum install -y vim net-tools wget bash-completion 。

密钥对准备

  • 在云服务器上通过 ssh-keygen 生成 RSA 密钥对 。
  • 后续我们需要将此公钥分发给本地 7 台机器，实现“云对地”的单向信任管理

推荐部署顺序

  • 基础设施：搭建 Firewall，调通 SNAT，确保内网能上网。

  • 管理链路：建立 Cloud-Hub 到 Firewall 的 SSH 隧道，测试 Ansible 能否 Ping 通内网。

  • 核心后端：部署 NAS (NFS) 和 MySQL，这是集群的“血液” 。

  • 业务前台：部署 Web1/2 和 LB 负载均衡 。

  • 全局观测：最后部署 监控体系，通过 Grafana 验证压力测试下的性能

集群架构-混合云1+7架构

我将架构分为 “云端控制平面” 和 “本地数据中心（IDC）” 两部分
核心逻辑为：“云端控制，本地生产，存储分离，安全加固”

云端控制平面（1 台云服务器）

节点名称 操作系统 核心服务 分工逻辑
Cloud-Hub Rocky 9 Ansible, Prometheus, Grafana, DNS, 堡垒机 中枢： 负责流量监控看板展示、下发自动化脚本、作为内网唯一的 SSH 入口 。

本地数据中心（7 台虚拟机，全 Host-Only 隔离）

节点名称 操作系统 核心服务 分工逻辑
Firewall Rocky 9 nftables (SNAT/DNAT) 门户： 负责内外网地址转换，开启路由转发 。
LB1 (M) Rocky 9 LVS + Keepalived 调度： 接收来自防火墙的 VIP 流量，主节点 。
LB2 (B) Rocky 9 LVS + Keepalived 调度： 接收来自防火墙的 VIP 流量，备节点 。
Web1 Rocky 9 Nginx 计算： 挂载 NFS 共享，处理具体的网页请求 。
Web2 Rocky 9 Nginx 计算： 与 Web1 形成集群，实现负载均衡 。
NAS Rocky 9 NFS Server 存储： 专门存储网页源码、图片、用户上传文件 。
MySQL Rocky 9 MySQL 8.4 (MGR) 数据： 存储结构化业务数据，建议独立部署以保证 IO 。

混合云架构的数据流转

管理流

通过公网 SSH 到 云端 Cloud-Hub -> 通过云端的 Ansible 穿透防火墙的 DNAT 端口 -> 批量管理本地的 7 台虚拟机 。

监控流

本地 7 台机器的 node_exporter 采集数据 -> 通过内网网关（Firewall）上报给 云端 Prometheus -> 云端 Grafana 实时绘图

业务流

外部用户 -> 本地 Firewall (WAN) -> LB 集群 (VIP) -> Web 集群 -> NAS/MySQL 。
