Docker概念_命名空间

发表于2026-03-09|更新于2026-03-09|Docker
|浏览量:

Docker 并不是一种全新的虚拟化技术，它本质上是一个利用 Linux 内核特性实现的隔离进程。

Docker 之所以能让容器看起来像一个独立的操作系统，全靠 Namespaces（资源隔离） 和 Cgroups（资源限制） 这两项核心技术。

基本定义

命名空间（Namespaces） 是 Linux 内核的一个特性，用于对内核资源进行物理隔离。

  • 通俗解释：想象你在一栋办公楼里。命名空间就像是给每个团队分配了一个独立的办公室。虽然大家都在同一栋楼（同一个 Linux 内核）里，但 A 团队看不见 B 团队的桌面，也听不见他们的电话。

  • 运维意义：它保证了容器 A 里的进程编号、网络配置、挂载点等，不会与容器 B 或宿主机发生冲突。

七大命名空间

PID Namespace-进程隔离

NET Namespace-网络隔离

MNT Namdespace-挂载点隔离

UTS Namdespace-主机名隔离

IPC Namdespace-进程间通信隔离

USER Namespace-用户隔离

CGROUP Namdespace
