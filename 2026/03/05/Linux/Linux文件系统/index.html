硬盘

与文件系统相关的硬件知识

在学习文件系统之前，先了解一下相关的硬件知识

存储介质-HDD&SSD

HDD

  • 盘片 (Platter) 与磁头 (Head)：数据通过磁头感应盘片上的磁性变化来读写。
  • 物理瓶颈：寻道时间 (Seek Time)。当文件系统请求一个碎片文件时，磁头必须物理移动到对应的磁道，盘片必须旋转到对应的扇区。
  • 文件系统偏好：HDD 极其厌恶“碎片”。因此，像 EXT4 或 XFS 这样的文件系统会尽量把同一个文件存在连续的物理空间上

SSD

SSD 内部没有机械零件，全是闪存颗粒（NAND Flash）

  • 读写单元：最小单位是 Page（页，通常 4KB-16KB），但擦除单位是 Block（块，包含多个 Page）。
  • 物理瓶颈：写放大 (Write Amplification)。SSD 不能直接覆盖数据，必须先擦除再写入。
  • 文件系统偏好：SSD 喜欢 TRIM 指令。文件系统删除文件时，必须通知硬件“这块地不用了”，方便硬件后台回收，否则 SSD 会越用越慢。

文件系统核心职能

硬盘本质上只能识别扇区（Sector）和块（Block）。文件系统在硬件之上建立了一个逻辑层：

  • 数据结构化： 它记录了某个文件（比如 我的简历.docx）具体占用了硬盘上的哪些物理位置。
  • 元数据管理： 除了文件内容，它还记录了文件的名字、大小、创建时间、修改权限（谁能看，谁能改）。
  • 空间分配： 当你保存新文件时，它负责找出一块空白区域安置数据，并标记该区域“已占用”。

文件系统三要素

  • 目录树（Directory Tree）： 这是我们最熟悉的界面。它允许我们用 /Users/Desktop/Photo.jpg 这种层级路径来定位文件，而不是记“磁道 5，扇区 20”。
  • 索引节点（Inode / File Table）： 这是文件系统的“后台账本”。每个文件都有一个唯一的编号，记录了文件属性和数据块的索引地址。
  • 超级块（Superblock）： 这是整个分区的“大脑”，记录了文件系统的整体信息（如：总容量多大、还剩多少空间、文件系统类型是 NTFS 还是 FAT32）。
