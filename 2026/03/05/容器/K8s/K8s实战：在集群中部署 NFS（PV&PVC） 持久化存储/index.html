K8s实战：在集群中部署 NFS 持久化存储

环境说明

  • 操作系统：CentOS 10 (Stream/Linux)
  • K8s 版本：v1.35 (最新特性支持)
  • 节点信息：
    • Master (NFS Server) | k8s-master-1 | 192.168.10.152
    • Worker Node | k8s-node-1 | 192.168.10.153
    • Worker Node | k8s-node-2 | 192.168.10.154
  • 镜像来源：由于网络限制，统一从“渡渡鸟”镜像站拉取。

集群准备：安装nfs客户端

所有 Node 节点必须安装 nfs-utils，否则 Kubelet 无法挂载卷

K8s资源层：PV与PVC的申领

  • PV (PersistentVolume)：由运维人员定义，描述真实的存储后端。

错误排查

在部署过程中，如果 Pod 处于 Pending 或 ContainerCreating，请按此清单排查：

  • PVC 找不到 (Not Found)：
    • 现象：Describe Pod 提示 pvc "..." not found。
    • 排查：确认 PVC 和 Pod 是否在同一个 namespace。
    • 修复：kubectl get pvc -A 检查名称和空间。
  • 无法绑定 (Pending)：
    • 现象：PVC 始终 Pending。
    • 原因：PV 和 PVC 的 accessModes 或 storageClassName 不匹配。
    • 修复：修改 YAML 使其完全一致。
  • 挂载超时 (Mount Failed)：
    • 原因：通常是 Worker Node 没装 nfs-utils 或防火墙拦截。
    • 修复：在 Node 节点执行 mount -t nfs 192.168.10.152:/data/k8s_nfs /mnt 手动测试网络连通性。
