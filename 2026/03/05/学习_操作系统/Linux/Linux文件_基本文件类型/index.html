Linux文件_基本文件类型

发表于2026-03-05|更新于2026-04-19|操作系统Linux
|浏览量:

前言

在 Linux 系统中，“一切皆文件”是其核心哲学。这意味着无论是硬件设备、目录还是进程通信，都被抽象为文件
为了区分这些文件，Linux 使用了不同的文件类型标识。可以通过执行 ls -l 命令查看文件权限位的第一个字符来识别它们

文件类型速查表

字符 文件类型 英文名称 描述
- 普通文件 Regular File 存储文本、数据或程序
d 目录文件 Directory 存放文件清单的特殊文件
l 软链接 Symbolic Link 指向另一个文件的指针
b 块设备 Block Device 磁盘、闪存等硬件
c 字符设备 Character Device 键盘、终端等顺序读写设备
p 管道文件 Named Pipe 进程间通信的 FIFO 通道
s 套接字 Socket 网络或本地服务通信接口

普通文件

从操作系统的底层视角来看，普通文件是一系列无结构的数据流。Linux 内核并不关心这个文件里存的是代码、文本还是猫片，它只负责维护其属性并提供读写接口。

核心特征

  • 文件标识符： 在 ls -l 的输出中，第一个字符为 -。
  • 存储位置： 数据实际存储在磁盘的“数据块”（Data Blocks）中，而文件的元数据（权限、大小、创建时间等）存储在 inode 中。
  • 无强制格式： 文件的内部结构由打开它的应用程序决定

逻辑分类

虽然内核一视同仁，但在用户应用层面，我们通常将其分为两大类

文本文件 (Text Files)

  • 内容： 由 ASCII 或 Unicode (UTF-8) 字符组成
  • 特点： 可以直接用 cat、vim 或 less 阅读
  • 示例： 配置文件 (/etc/passwd)、源代码 (hello.c)、日志文件 (/var/log/syslog)。

二进制文件 (Binary Files)

  • 内容： 包含机器可读的代码或压缩后的特定格式数据。
  • 特点： 直接打开会显示乱码。
  • 示例： * 可执行程序： 如 /bin/ls（ELF 格式）。
    • 压缩包： .tar.gz, .zip。
    • 多媒体： 图片 (.png)、视频 (.mp4)。

目录文件

目录文件 (Directory) 并不是一个存放文件的“文件夹”容器，而是一种特殊的普通文件

本质

在底层，目录文件存储的内容非常简单，它是一张映射表（Mapping Table）。每一条记录（称为 dentry，即 Directory Entry）主要包含两个核心信息：

  • 文件名 Filename
  • Inode编号 Inode Number
    当你执行 ls /home/user 时，系统并不是去“打开文件夹”，而是读取 /home/user 这个目录文件里的内容，查看到底有哪些文件名和对应的 Inode 号。

每个目录下至少包含两个特殊的子目录：

  • . (当前目录)：指向目录自身的 Inode。
  • .. (上级目录)：指向父目录的 Inode。

标识符与基本特征

  • 标识符： 在 ls -l 输出的第一个字符为 d。
  • 大小： 目录文件也有大小（通常是 4096 字节的倍数）。这个大小反映的是文件名列表占用空间的大小，而不是目录内所有文件的总大小。
  • Inode 占用： 每个目录都有自己的 Inode，记录了它的权限、所有者和修改时间。

权限位的特殊含义

对于目录文件，rwx 权限的含义与普通文件完全不同：

权限 对目录的含义 如果没有该权限…
r (Read) 读取目录列表 你无法执行 ls 查看目录下有哪些文件。
w (Write) 修改目录内容 你无法在该目录下创建、删除或重命名文件（即使你有文件的写权限）。
x (Execute) 进入目录 你无法 cd 进去，也无法访问该目录下的文件内容（即使你知道文件名）。

如果一个目录只有 x 权限而没有 r 权限，你不能 ls 看到文件名，但如果你精确知道某个文件名（如 secret.txt），你依然可以 cat 它。

链接文件

链接文件 (Link File) 是指向文件系统中另一个对象的“引用”。
Linux 提供了两种截然不同的链接方式：

  • 软链接（Symbolic Link）
  • 硬链接（Hard Link）。
    理解它们的区别是掌握 Linux 文件系统的关键。

软连接-Symbolic Link / Soft Link

这是我们在 ls -l 中看到的以 l 开头的文件。它更像是一个独立的文件，其内容仅仅是另一个文件或目录的路径字符串。

  • 标识符： l
  • Inode 特性： 软链接拥有自己独立的 Inode。
  • 跨分区： 可以跨越不同的磁盘分区或文件系统。
  • 目录链接：允许为目录创建软链接。
  • 失效情况： 如果原始文件被删除，软链接会变成“死链接”（Broken Link），指向一个不存在的路径。

硬链接-Hard Link

硬链接在 ls -l 中并不表现为特殊类型（它看起来就是普通文件 -），但它是文件系统层面的一个重要概念。

  • 本质： 它是同一个 Inode 的另一个名字。
  • Inode 特性： 硬链接与原始文件共享同一个 Inode 编号和相同的数据块。
  • 引用计数： 每增加一个硬链接，该 Inode 的 Links 计数就会加 1。只有当计数降为 0 时，文件数据才会被真正删除。
  • 限制： * 不能跨文件系统（因为不同分区的 Inode 编号可能冲突）。
    • 出于安全和防止环路的考虑，通常不允许用户为目录创建硬链接。

对比

特性 软链接 (Symbolic) 硬链接 (Hard Link)
命令 ln -s source link ln source link
Inode 不同的 Inode 相同的 Inode
文件大小 路径字符串的大小 与原文件大小完全一致
跨分区 支持 不支持
链接目录 支持 不支持
删除原文件 链接失效（红字闪烁） 文件依然可用，计数减 1

块设备文件

块设备文件 (Block Device File) 是用于与存储硬件进行高效、随机访问交互的特殊文件
它是 Linux “一切皆文件”哲学的典型体现：你不需要通过复杂的底层指令去读写硬盘，只需像操作普通文件一样 open、read、write 这个设备文件即可

核心特征

  • 标识符： 在 ls -l 的输出中，第一个字符为 b。
  • 随机访问 (Random Access)： 这是块设备最本质的特征。你可以直接跳转到文件的任何位置（使用 lseek）读取数据，而不需要按顺序从头读到尾。
  • 缓冲机制 (Buffering)： 内核为块设备维护了一个特殊的“缓冲区高速缓存”（Buffer Cache）。当你写入数据时，内核通常先将其存入内存缓存，待时机成熟后再批量写入物理磁盘，以提高性能。
  • 数据单位： 数据以固定大小的“块”（Block，如 512 字节、1024 字节或 4096 字节）为单位进行传输。

常见的块设备示例

块设备几乎涵盖了所有的持久化存储介质，通常存放在 /dev 目录下：

  • /dev/sda, /dev/sdb：SATA/SAS 硬盘或 SSD。
  • /dev/nvme0n1：现代 NVMe 固态硬盘。
  • /dev/mmcblk0：SD 卡或 eMMC 存储。
  • /dev/loop0：回环设备（将一个普通镜像文件模拟成块设备）。

主设备号与次设备号

不同于普通文件记录大小，ls -l 查看块设备时，原本显示“大小”的位置显示的是两个数字：

  • 8 (主设备号)： 标识设备驱动程序。内核通过这个数字知道该找哪个驱动来处理请求（8 通常代表 SCSI/SATA 硬盘驱动）。
  • 0 (次设备号)： 标识该驱动程序下的具体设备实例（0 代表第一块硬盘，1 代表第一个分区等）。

块设备与文件系统的关系

这是一个常见的误区：块设备文件本身并不等同于文件系统。

  • 块设备： 是“原始的存储空间”（Raw Storage），像一块白纸。
  • 文件系统： 是在块设备上“画好的格子”（如 Ext4, XFS）。 你需要通过 mkfs 命令在块设备上创建文件系统，然后通过 mount 将其挂载到某个目录。

字符设备

字符设备 (Character Device) 是与硬件进行通信的另一种主要方式。

核心特征

  • 标识符： 在 ls -l 输出的第一个字符为 c。
  • 顺序访问 (Sequential Access)： 数据像流水一样按顺序读取或写入。与块设备不同，字符设备通常不支持随机寻址（你不能像在硬盘上那样直接跳转到某个位置读取，除非驱动特殊实现）。
  • 无缓冲 (Unbuffered)： 字符设备通常直接与硬件驱动交互，不经过内核的页缓存（Page Cache）。你写一个字节，驱动就立刻处理一个字节。
  • 单位： 以字节（Byte）为单位进行传输。

常见的字符设备示例

字符设备涵盖了大多数非存储类的外设

  • /dev/tty, /dev/pts/*：终端和伪终端。
  • /dev/lp0：打印机。
  • /dev/mouse, /dev/input/event*：鼠标、键盘等输入设备。
  • /dev/console：系统控制台。
  • /dev/null, /dev/zero, /dev/urandom：这些是虚拟字符设备（软件模拟的设备）。

特殊的虚拟字符设备

在 Linux 系统维护中，这几个字符设备最为权威且常用

  1. /dev/null：位桶（Bit Bucket）。写入它的任何数据都会消失，读取它会立即返回 EOF（文件结束）。
  2. /dev/zero：无限零源。读取它会得到无穷无尽的 \0。
  3. /dev/random / /dev/urandom：内核随机数生成器接口。

主设备号与次设备号

与块设备一样，字符设备也使用一对数字来标识：

  • 1 (Major)：标识字符设备驱动程序（1 通常是内存设备驱动）。
  • 3 (Minor)：标识该驱动下的具体功能（3 代表空设备 null）。

字符设备与块设备的本质区别

特性 字符设备 (Character) 块设备 (Block)
数据访问 按字节流，顺序访问 按块，支持随机访问 (lseek)
内核缓冲 通常无缓冲 有 Buffer Cache 缓冲
典型硬件 键盘、串口、声卡 硬盘、U 盘、光驱
ls -l 标识 c b

管道文件

管道文件 (Pipe) 是进程间通信（IPC, Inter-Process Communication）的最基本方式。它就像一根单向传输的数据管子，数据从一头流入，从另一头流出。
管道分为两种：匿名管道和有名管道 (Named Pipe/FIFO)。通常我们所说的“管道文件”特指有名管道。

核心特征

  • 标识符： 在 ls -l 输出的第一个字符为 p。
  • FIFO 原则： 遵循“先进先出”（First In, First Out）。先写入管道的数据一定先被读出来。
  • 单向性： 它是半双工的，数据只能在一个方向上流动。如果需要双向通信，通常需要建立两个管道。
  • 阻塞特性（重要）： * 如果进程尝试读取一个空的管道，它会阻塞（暂停），直到有数据写入。
    • 如果进程尝试写入一个已满的管道，它也会阻塞，直到有空间释放。

有名管道（FIFO）&匿名管道

特性 匿名管道 (Anonymous Pipe) 有名管道 (Named Pipe / FIFO)
存在形式 只存在于内核内存中，无文件名 在文件系统中可见，有路径名
使用方式 命令行中的竖线 ` `
进程关系 仅限具有亲缘关系（父子）的进程 任意进程之间均可通信
生命周期 随进程结束而销毁 除非手动删除，否则文件一直存在

套接字文件

套接字文件 (Socket) 是实现进程间通信（IPC）最强大的方式。如果说管道是简单的“水管”，那么套接字就是一套完整的“邮政系统”。
它是 Linux 中唯一一种能跨越网络、在不同物理机器之间进行通信的文件类型。

核心特征

  • 标识符： 在 ls -l 输出的第一个字符为 s。
  • 双向通信 (Full-Duplex)： 与单向的管道不同，套接字支持同时收发数据。
  • 多样性： 既支持同一台机器上的本地通信（Unix Domain Sockets），也支持跨网络的 TCP/IP 通信。
  • 文件化表现： 虽然网络连接本身很复杂，但 Linux 将其抽象为一个文件。你可以通过文件描述符来读写它。

本地套接字

你在文件系统中看到的 s 类型文件，通常是 Unix Domain Socket

  • 用途： 专门用于同一台主机上的进程间通信。
  • 优势： 因为不需要经过网络协议栈（如 TCP/IP 校验、封包等），它的速度比网络套接字快得多。
  • 常见例子：
    • /var/run/docker.sock：Docker 守护进程的通信接口。
    • /tmp/mysql.sock：MySQL 数据库的本地连接文件。

套接字的工作流程

在 Linux 编程手册 (man 2 socket) 中，创建一个套接字通信通常需要以下步骤：

  1. socket()：创建一个新的套接字，返回一个文件描述符。
  2. bind()：将套接字绑定到一个“地址”。对于本地套接字，这个地址就是一个文件路径；对于网络套接字，它是 IP+端口。
  3. listen()：服务器端进入监听状态。
  4. accept()：接收客户端的连接请求。
  5. read() / write()：像操作普通文件一样交换数据。

网络套接字

虽然它们也是套接字，但网络套接字通常不会在文件系统中显示为 s 类型的实体文件。它们存在于内核内存中，通过网络协议栈直接交互

  • 查看方法： 使用 ss 或 netstat 命令。

安装方式在CentOS 10 环境下，不需要安装整个 Apache 服务，只需要安装工具包即可 1yum install -y httpd-tools 核心参数ab 的参数非常多，但作为运维，你只需要死记硬背下面这四个： -n (Number)：总请求数。即一共要发多少次请求。 -c (Concurrency)：并发数。即模拟多少个人同时在点你的网站。 -t (Timelimit)：最大测试时间。如果请求没发完但时间到了，测试也会停止。 -k (KeepAlive)：开启长连接。这更符合现代浏览器的行为，可以减轻频繁建立 TCP 连接的开销。 实战演练
