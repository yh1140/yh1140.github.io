Key通用命令

在 Redis 中，无论 Value 是什么类型，对 Key 的管理是通用的。这是 SRE 维护内存空间最常用的手段

命令 作用 SRE 笔记
SET key value 存入一个键值对 最基础的开始
EXISTS key 判断键是否存在 返回 1 存在，0 不存在
TYPE key 查看键的数据类型 排查数据结构不一致时必用
DEL key 删除键 $O(1)$ 操作，但在删大 Key 时会阻塞
EXPIRE key 60 设置 60 秒后过期 SRE 核心技能： 严禁不设过期的缓存！
TTL key 查看剩余寿命 返回 -1 永不过期，-2 已过期/不存在

五大基本数据结构的简答操作

1. String（字符串）- 简单的存取

2. List（列表）- 像排队一样

3. Hash（哈希）- 存结构化对象

4. Set（集合）- 去重容器

5. ZSet（有序集合）- 带权重的排行榜

这些操作要谨慎

  • FLUSHALL / FLUSHDB：清空所有数据。

  • KEYS *：遍历所有 Key。如果内存里有千万级数据，Redis 会直接卡死，导致业务全线崩溃。

  • 大 Key 删除：如果你直接 DEL 一个包含百万元素的 List，Redis 也会
