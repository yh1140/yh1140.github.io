python进阶_生成器

发表于2026-04-03|更新于2026-04-07|Python编程语言
|浏览量:

生成器（Generator） 是 Python 中一种极其高效且优雅的特殊迭代器。它允许你定义一个能够逐个产生值的逻辑，而不是一次性将所有值存储在内存中。

我们可以从定义、核心机制、运行逻辑和优势四个维度来系统地理解它。

形式定义

在 Python 中，只要一个函数体内包含了 yield 关键字，这个函数就不再是一个普通函数，而是一个生成器函数。

  • 普通函数：调用后立即会自行，遇到 return 或执行完毕后返回结果，并销毁内部所有局部变量。
  • 生成器函数：调用后不会立即执行函数体，而是返回一个生成器对象。只有当你在循环中迭代它，或者手动调用 next() 时，它才会开始执行。

核心机制-暂停与状态保持

生成器的精髓在于 “暂停执行”

  • yield 的作用：它相当于一个“路障”兼“临时出口”。当程序运行到 yield 时，它会做两件事：
    1. 将 yield 后面的值返回给调用者。
    2. 原地封存当前函数的所有状态（包括所有局部变量、当前的执行位置、异常处理状态等）。
  • 恢复执行：当下一次请求值时（例如 for 循环的下一轮），生成器会从上次 yield 的下一行代码开始继续执行，就像从未停止过一样。

生成器vs普通函数

特性 普通函数 (return) 生成器函数 (yield)
返回方式 一次性返回所有结果 逐个产生结果
内存占用 随结果数量线性增加 始终保持极低（常量级）
生命周期 执行完即销毁状态 产生完最后一个值才销毁
执行模式 贪婪模式（一次算完） 惰性模式（要一个算一个）
