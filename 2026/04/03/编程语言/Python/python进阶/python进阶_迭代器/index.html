python进阶_迭代器

  • 而是为了”生成”或”流式处理”数据

可迭代对象-定义

从系统架构和逻辑设计的角度来看，可迭代对象（Iterable） 并不是某种特定的数据类型，而是一种对象行为的协议（Protocol）。

逻辑关系

从抽象逻辑来看，可迭代对象是一个产生迭代器的工厂。

  • 可迭代对象不一定负责记住当前遍历到哪了
  • 但是它必须保证能够提供一个工具（迭代器）来完成遍历
  • 所有的迭代器都是可迭代对象
  • 可迭对象不一定是迭代器
    $$Iterators \subset Iterables$$

底层协议

在 Python 的鸭子类型（Duck Typing）设计中，一个对象是否是可迭代的，取决于它是否遵循以下迭代协议之一：

  • 显示协议__iter__方法
    对象实现了 __iter__() 方法。当你调用 iter(obj) 时，系统会执行该方法并期望返回一个迭代器对象。
  • 兼容协议__getitem__方法
    如果对象没有 __iter__，但实现了 __getitem__（支持按索引访问），并且能够处理从 0 开始的整数索引，系统在逻辑上仍视其为可迭代对象。

迭代的执行逻辑

当你写下 for item in iterable: 这行代码时，系统在底层经历了一个标准的逻辑闭环：

  • 初始化阶段-调用内置函数iter(iterable)
    • 如果对象有__iter__，返回其自动生成的迭代器
    • 如果只有__getitem__，系统自动船舰一个适配迭代器
  • 提取阶段
  • 终止阶段
  • 异常处理

可迭代对象

  • 定义：实现了 __iter__ 方法的对象，或者实现了 __getitem__ 方法且支持序号访问的对象。
  • 逻辑本质：它代表一个“数据集合”，但它本身不负责记录遍历的进度。
  • 常见例子：list, tuple, dict, set, str。

迭代器

  • 定义：同时实现了 __iter__ 方法和 __next__ 方法的对象。
  • 逻辑本质：它是一个“带状态的游标”。它不仅知道数据在哪里，还知道“下一个”是谁。
  • 特性：迭代器一定是可g 迭代对象，但反之不成立。

可迭代协议

可迭代协议是第一层契约，它定义了哪些对象可以被放在 for 循环的右侧。

  • 契约要求：对象必须实现 __iter__ 方法。
  • 行为逻辑：当 Python 执行 iter(obj) 时，它会调用 obj.__iter__()。该方法的目标不是返回值，而是返回一个“执行者”（即迭代器）。
  • 形象比喻：可迭代对象就像是一本书。书本身不能自动翻页，但它提供了一个“翻页动作”，通过这个动作你可以找一个“书签（迭代器）”来帮你记录读到了哪。

补充逻辑：如果一个对象没有 __iter__ 但实现了 __getitem__（支持按索引访问），Python 也会尝试将其视作可迭代对象。

迭代器协议

迭代器协议是第二层契约，它定义了如何从一个序列中逐个获取元素。

  • 契约要求：对象必须同时实现以下两个方法：
    1. __next__：返回序列中的下一个元素。如果没有元素了，必须抛出 StopIteration 异常。
    2. __iter__：返回迭代器本身（即 return self）。
  • 行为逻辑：迭代器是一个“有状态”的对象，它记录了当前的遍历位置。
  • 形象比喻：迭代器就像是那个“书签”。它知道当前在哪一页（状态），你每拨动它一次（调用 next()），它就告诉你这一页的内容并跳到下一页。

迭代器的四大特性

惰性计算

  • 最强的特性
  • 只有被请求（__next__()）时才会计算或返回下一个值
  • 而不是一次性把所有数据加载到内存
  • 处理海量数据时占用内存极低

一次性

  • 迭代器单向行进，只能向前，不能向后
  • 一旦一个迭代器被耗尽，抛出StopIteration，他就废弃了
  • 只能重新调用iter()获取一个新的迭代器对象

内存效率

  • 对比列表和迭代器
    • List：内存复杂度O(n)
    • Iterator：仅存储当前状态，内存复杂度O(n)

支持无限序列
