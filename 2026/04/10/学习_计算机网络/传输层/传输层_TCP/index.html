TCP-Transmission Control Protocol传输控制协议
一种可靠的传输控制协议
保证数据：

  • 不丢失
  • 不错位
  • 不重复
  • 可校验

核心定义

TCP 是位于 传输层 (Layer 4) 的协议。如果说 IP 协议负责把数据包从地址 A 送到地址 B（类似快递员），那么 TCP 就负责确保包裹完整、按顺序送达，且在丢包时要求重发。

最大的特点：

  • 面向连接
  • 可靠
  • 基于字节流

核心机制

  • 序列号+确认应答+超时重传
  • 面向连接
  • 滑动窗口
  • 流量控制
  • 拥塞控制
  • 快速重传
  • 去重处理
  • 按序交付
  • 校验机制

TCP头部结构

TCP头部不包含选项信息时，默认20字节的长度

选项

三次握手

四次挥手

①主动关闭方->被动关闭方

  • 主动方发送FIN = 1, seq = u表达数据发送完成，请求关闭连接
  • 主动方进入FIN-WAIT-1

②被动关闭方->主动关闭方

  • 被动方收到后，回复ACK = 1, seq = v, ack = u+1告诉发送方收到请求，并请求主动方等待
  • 被动方进入CLOSE-WAIT状态
  • 主动方收到回复之后进入FIN-WAIT-2状态，等待被动方发送最后的FIN
  • 此时连接处于半关闭的状态。主动方不能发数据，只能收数据

③被动关闭方->主动关闭方

  • 被动方数据发完了，发送FIN = 1, ACK = 1, seq = w, ack = u+1

④主动关闭方->被动关闭方
