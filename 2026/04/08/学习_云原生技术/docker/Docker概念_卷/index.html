Docker概念_卷

发表于2026-04-08|更新于2026-04-08|容器Docker
|浏览量:

在 Docker 的世界里，有一个核心原则：容器是易失的（Ephemeral）。这意味着容器一旦被删除，其内部产生的所有数据都会随之烟消云散。
为了解决数据持久化以及容器间共享数据的问题，Docker 引入了**卷（Volumes）**的概念

Docker卷操作手册

为了解决此问题，docker提供了三种方式将数据挂载到容器中

  • Vollumes
    • 存储位置： 由 Docker 管理，在 Linux 上通常位于 /var/lib/docker/volumes/。
    • 特点： 这是 官方推荐 的方式。你不需要关心它在宿主机具体的物理路径，一切由 Docker 托管。
    • 优点：
      • 完全由 Docker API 管理，更安全。
      • 支持存储驱动（可以挂载到云存储或 NFS）。
      • 容器之间共享非常方便。
  • Bind Mounts
    • 存储位置： 宿主机上的 任意路径（如 /home/user/app/config）。
    • 特点： 这种方式非常依赖宿主机的目录结构。
    • 优点： 适合开发阶段。你可以把代码目录挂载进去，宿主机修改代码，容器里实时生效。
    • 缺点： 移植性差，如果宿主机路径变了，容器就跑不起来。
  • tmpfs Mounts
    • 存储位置： 宿主机的 系统内存 中。
    • 特点： 非持久化。
    • 优点： 读写极快，适合存放敏感信息（不希望写在磁盘上）或临时缓存。
特性 Volumes Bind Mounts tmpfs
宿主机路径 Docker 自动创建/管理 用户指定任意路径 仅内存
可移植性 高（不依赖特定路径） 低（依赖宿主机路径） 中
管理方式 Docker CLI 命令管理 操作系统文件管理 N/A
初始内容 如果卷为空，容器内内容会复制到卷 容器内内容会被覆盖 初始为空
使用场景 持久化存储 写代码调试 极致性能

如果没有卷，会面临以下几个问题

  • 数据随删随丢，容器一旦崩溃，数据就全丢了
  • 写入性能差，容器层（Writable Layer）的文件系统通过联合文件系统（UnionFS）实现，在大规模写入时效率较低。
  • 数据耦合： 数据存储在容器内部，很难在不停止容器的情况下提取或迁移。

卷_概念

简单来说，卷（Volume）是 Docker 官方提供的、独立于容器生命周期的“数据保险箱”，专门用于持久化和共享那些不随容器销毁而丢失的数据。

卷_特性

概括：Docker 实现数据持久化和容器间数据共享的核心机制，也是避免容器数据随容器销毁而丢失的关键技术。在启动容器时

核心定义：由 Docker 管理的、独立于容器生命周期的持久化数据存储机制。卷本质上是宿主机文件系统中的一个特殊目录（默认位于 /var/lib/docker/volumes/），它可以被一个或多个容器挂载使用，容器对卷的读写操作与直接操作本地文件系统一致

特性：

数据持久化

容器间数据共享

宿主机与容器数据解耦

性能更优且兼容性更强

###docker卷类型###

##命名卷Named Volume[受管理卷]

#无用户自定义名称，由 Docker 生成随机唯一长字符串作为标识；无需手动创建，运行容器时自动生成；仅能通过容器关联找到

#存储位置：/var/lib/docker/volumes/<随机唯一ID>/_data

#使用场景：临时数据持久化、容器内部自动挂载、无需长期保留的临时数据（如应用缓存、临时日志）

#管理难度：高（难以识别对应业务，不利于复用、备份和维护）

#可移植性：高（Docker 统一管理，可随容器迁移）

##匿名卷Anonymous Volume[受管理卷]

#有用户自定义的显式语义化名称；可手动创建，也可在运行容器时自动创建；易于识别和关联业务

#存储位置：/var/lib/docker/volumes/<自定义卷名>/_data

#使用场景：长期数据持久化（如数据库、核心配置）、容器间数据共享、需要频繁复用和备份的卷

#管理难度：低（易识别、易管理、易复用，生产环境首选）

#可移植性高（Docker 统一管理，支持导出 / 导入迁移）

##绑定挂载Bind Mount

#非 Docker 管理，由用户手动指定宿主机的任意目录 / 文件挂载到容器内；需显式指定宿主机路径（绝对路径优先）

#存储位置：宿主机任意自定义路径（如 /home/app/data或./html）

#使用场景：开发场景（本地代码目录挂载到容器，实时同步修改）、宿主机已有数据映射到容器

#管理难度：中（需自行维护宿主机路径权限、目录结构，路径错误会导致挂载失败）

#可移植性：低（依赖宿主机目录结构，更换环境需重新配置路径）
