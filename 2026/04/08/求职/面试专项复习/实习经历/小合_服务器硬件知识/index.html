服务器硬件概述

1. 服务器硬件的组成与功能

服务器本质上是一台高性能计算机，但其设计核心在于稳定性、可扩展性和冗余性。

  • 计算组件：CPU、内存（RAM）。
  • 存储组件：硬盘（HDD/SSD）、RAID卡、NVMe 推进器。
  • 网络组件：网卡（NIC）、光纤通道卡（HBA）。
  • 支撑组件：主板、电源（通常为 1+1 冗余）、散热风扇阵列。
  • 管理组件：远程管理芯片（BMC）。

2. 服务器的分类

  • 按外形分类：
    • 机架式 (Rack Server)：最主流，以 U 为单位（1U=4.445cm）。常见 1U、2U、4U。
    • 塔式 (Tower Server)：类似 PC，适用于无机房的小型办公环境。
    • 刀片式 (Blade Server)：高密度部署，共用电源和散热，适合大规模集群。
  • 按处理器架构分类：x86 服务器（Intel/AMD）、ARM 服务器（Ampere/鲲鹏）、机（IBM Power）。

CPU

1. 工作原理

CPU 遵循 取指（Fetch）-> 解码（Decode）-> 执行（Execute）-> 写回（Write-back） 的循环。服务器 CPU 更强调多任务处理能力和指令集的丰富性（如虚拟化指令集 VT-x）。

2. 核心性能参数

  • 主频 (Clock Speed)：单核运算速度。
  • 核心数/线程数 (Cores/Threads)：并行处理能力。
  • 缓存 (Cache)：L1/L2/L3 缓存。服务器 CPU 的 L3 缓存 通常很大，能极大地提升数据库等应用的性能。
  • TDP (热设计功耗)：决定了散热系统的设计需求。

3. CPU 架构与多核技

  • x86-64：目前的主流，Intel 至强（Xeon）和 AMD 霄龙（EPYC）。
  • NUMA 架构 (Non-Uniform Memory Access)：多路服务器中，CPU 访问本地内存快于访问远端内存。在 Linux 优化中，NUMA 绑定是提升性能的常见手段。
  • 超线程 (Hyper-Threading)：将一个物理核心模拟成两个逻辑核心。

RAM

1. 工作原理与 ECC 技术

服务器内存不仅用于存取数据，最重要的特性是 ECC（纠错码）。它能检测并修复单比特错误，防止系统因“位翻转”导致的蓝屏或宕机。

2. 类型与规格

  • DDR4 vs DDR5：DDR5 提供了更高的带宽和更低的功耗。
  • RDIMM (Registered DIMM)：带寄存器的内存，支持更高的容量和稳定性，是服务器的标准配置。
  • 通道 (Channel)：服务器主板通常支持 8 通道 内存，必须插满或对称插入才能发挥最大性能。

存储

1. HDD 与 SSD

  • HDD (机械硬盘)：容量大、成本低，适用于冷数据存储。接口多为 SAS（比 SATA 更稳定、支持全双工）。
  • SSD (固态硬盘)：
    • SATA SSD：旧式接口，速度上限约 600MB/s。
    • NVMe SSD：走 PCIe 通道，速度可达数 GB/s，是高性能数据库的首选。

2. 性能查看与 RAID 技术

  • 测试工具：常用 fio 测试 IOPS（随机读写速度）和吞吐量（顺序读写速度）。
  • RAID 技术 (磁盘阵列)：
RAID 级别 最少磁盘 冗余性 性能表现 适用场景
RAID 0 1 无 极高（条带化） 临时数据
RAID 1 2 极高（镜像） 读快写慢 操作系统盘
RAID 5 3 中（校验位） 读快写均衡 通用存储
RAID 10 4 高（先镜像再条带） 极高 数据库核心业务

网卡

1. 网卡类型 (NIC)

  • 千兆/万兆 (1GbE/10GbE)：目前企业级主流已转向 25GbE 或 100GbE。
  • 电口 vs 光口：电口用超六类线，距离短；光口用光纤+光模块，传输距离远且抗干扰。

2. 远程管理 (IPMI/BMC)

这是服务器区别于 PC 的最大标志

  • BMC (基板管理控制器)：主板上的一块独立芯片。
  • 厂商实现：Dell 的 iDRAC、HPE 的 iLO、华为的 iMana/iBMS。
  • 功能：即便服务器处于关机状态，只要电源接通，你就能通过网页远程开机、挂载 ISO 安装系统、查看硬件温度。

主板

1. 主板 (Motherboard)

服务器主板通常有两个 CPU 插座（双路），分布着密集的内存插槽。

2. PCIe 扩展槽

  • 用于插入 RAID 卡、网卡、GPU 加速卡等。
  • 目前主流为 PCIe Gen 4/5，每一代的带宽都会翻倍。

服务器与运维故障处理

1. 上架与初始化

  • 导轨安装：确保理线架不阻碍散热。
  • BIOS 设置：开启虚拟化支持、设置能效模式（通常调为 Performance）、配置 RAID 阵列。

2. 操作系统安装

  • PXE 网络安装：大规模部署时，通过网络自动安装系统（结合 Kickstart 或 Terraform）。

3. 常见故障排查逻辑

遵循“由外向内、由易到难”：

  1. 面板指示灯：琥珀色通常表示硬件故障（硬盘坏了、电源挂了）。
  2. 查看日志：通过远程管理卡查看 SEL (System Event Log)。
  3. 最小化测试：拔掉冗余配件（如只留一根内存、一个 CPU），观察能否过 POST（上电自检）。
