底层环境虚拟化搭建

一、 采集器底层环境部署 (PVE 虚拟化)

该阶段侧重于基础设施的构建与安全加固，确保采集器主机具备稳定的运行环境。

  • 基础环境准备：登录 PVE 管理平台，确认网络为桥接模式，并强制执行高强度调改密策略。

  • 网络安全加固：通过 ufw 和 iptables 配置严谨的访问控制策略（ACL），仅允许 JumpServer 及公司特定管理 IP 访问相关端口（如 22、8000-9000 等），完成录入后及时清理临时规则。

  • 虚拟机构建：

    • 镜像处理：通过 wget 下载或 JumpServer 上传 qcow2 镜像。

    • 硬件分配：创建 4核/8G/200G 规格的虚拟机。

    • 磁盘导入：利用 qm importdisk 命令将镜像导入 local-lvm，并修正启动顺序与开机自启配置。

  • 系统初始化：配置与宿主机 PVE 同网段的静态 IP，确保内网互通。

二、 控制器系统初始化 (数据安全与存储)

该阶段重点在于控制器内部的逻辑配置，特别是针对数据存储的安全性处理。

  • LUKS 硬盘加密：使用 cryptsetup 对新挂载的数据盘进行物理加密（统一密码规范），防止底层数据外泄。

  • 分区与挂载：完成加密分区的映射（LuksOpen）与 ext4 文件系统格式化，并挂载至指定目录 /nolinux。

  • 权限对接：再次进行防火墙加固并接入 JumpServer 统一管理。

三、 sFlow 业务配置实操 (流量采集)

该阶段由系统配置转向业务执行，实现流量数据的有效抓取。

  • 仿真与对接：利用 eNSP 模拟器绘制拓扑，验证虚拟机与交换机间的采集路径。

  • 交换机配置：在物理/虚拟交换机上开启 sFlow 代理，配置采集器 IP、端口及采样率等核心参数。

四、 运维分析与流量结算 (系统应用)

该阶段指导员工如何利用 Web 系统进行日常运维与业务核算。

  • Web 系统操作：掌握各项参数调整方法及使用说明。

  • 故障排查逻辑：利用系统“源 IP 查找”与“业务分析”功能，定位未进调度节点的异常流量，检查默认路由下的残留流量。

  • 流量结算：基于 sFlow 采集的真实数据实现业务侧的流量结算与统计。
