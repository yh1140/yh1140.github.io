腾讯服务器部署指南

一、磁盘分区

1. 查看硬盘的情况

 

根据硬盘的具体大小进行分盘  

机械盘不需要进行分区（如sdb sdc等）

Ssd固态盘需要进行分区(总分区数大于等于6）：

1. 建议设备上所有SSD数据盘都进行分区

2. SSD盘进行等大小分区，单分区≥1TB，如4T SSD盘可分为4个大小相同的1T分区

3. （并非严格要求1T，但是分区大小需要相同，比如对于3.2TB磁盘，单分区大小800G也符合要求）

不要动系统盘（sda！！！！！在提交验收时需要注意系统盘的目录是否正确）

2. 给固态盘分区                                    

fdisk /dev/nvme0n1  # 进入nvme0n1 进行分区(g，w；；n，回车，+7800g（一直新建）；；)

按 g 改为gpt模式

按 n 开始分区

从1开始分区

d删除分区

 

反复执行之前的操作  最后按w 保存推出

二、给分区之后的硬盘进行格式化

mkfs.xfs -f /dev/sdb1

mkfs.xfs -f /dev/nvme0n1p1

三、创建需要挂载的目录，并且挂载到磁盘

mkdir /pcdn_data/  ###固态盘目录

mkdir /pcdn_data_hdd/ ##机械盘目录    

（在根目录，创建两个文件夹mkdir pcdn_data、 mkdir pcdn_data_hdd （ssd、hdd的父目录）

mkdir /pcdn_data/storage1_ssd   ##从sdb - sdh 对应 storage1_ssd - storage7_ssd

 

mkdir /pcdn_data_hdd/storage1_hdd  ### 分区了几块盘就创建几个

 

四、把硬盘挂载到创建的文件夹里（临时挂载磁盘）

mount /dev/sdb1 /pcdn_data_hdd/storage1_hdd/  

mount /dev/nvme0n1p1 /pcdn_data/storage1_ssd/

五、在配置文件中挂载 （自动挂载，永久挂载磁盘）

vi /etc/fstab  //进入配置文件

/dev/sdb1 /pcdn_data_hdd/storage1_hdd xfs defaults 0 0

 

六、关闭selinux 在防火墙上开启8081的端口

1. 关闭selinux

vi /etc/selinux/config  //进入配置文件中修改

SELINUX=disabled

2. 关闭防火墙

systemctl stop firewalld  //关闭防火墙

systemctl disable firewalld

systemctl status firewalld  //查看防火墙状态

vi /etc/sysconfig/network-scripts/ifcfg-eth0   

//修改centos中对应网卡的ip地址的配置文件，把DNS1设置成为114.114.114.114

DNS1=114.114.114.114

service network restart  //重启network

**七、**查看硬盘大小的包

yum install -y smartmontools  //linux硬盘检测工具

smartctl –all /dev/nvme0n1

 

八、添加配置文件

1. 创建文件夹

mkdir /etc/pcdn/

2. 配置文件

vi /etc/pcdn/pcdn.conf

 

**九、**配置允许访问的ip

yum -y install xinetd  //安装成功后即可

service xinetd start/stop/restart     

//启动，关闭，重启ftp服务     

vi /etc/hosts.allow    

sshd:101.91.92.223:allow

sshd:122.192.127.45:allow

sshd:175.6.107.254:allow

sshd:106.52.223.213:allow

//一个IP请求连入，linux的检查策略是先看/etc/hosts.allow中是否允许，如果允许直接放行；如果没有，则再看/etc/hosts.deny中是否禁止，如果禁止那么就禁止连入

 

登录服务器，通过175.6.107.254跳转（root 密码Wlw@=%91^72!）

service xinetd restart

 ssh-keygen -R [服务器ip address]   //如果跳转不了，在公网上添加这条

Reboot

uname -a   //显示系统名、节点名称、操作系统的发行版本号、内核版本等等

连续ping包，看服务器有没有起来，如果没有起来，可能是挂载问题或者不稳定导致的现象，需联系机房那边

十五、修改snmp（做完十一的第1个点就先把这个做了）

1、 被监控机安装snmp（如有则不需要安装）

yum -y install net-snmp net-snmp-utils

2、被监控机的配置

vi  /etc/snmp/snmpd.conf

(1) 找到com2sec notConfigUser  default       public 
改为：com2sec notConfigUser   192.168.1.1（改成监控服务器的ip，也就是cacti所在服务器的ip）       public 

 

(2) 找到access  notConfigGroup “”      any       noauth    exact  systemview  none none 
改为：access  notConfigGroup “”      any       noauth    exact  all  none  none 

(3)、找到#view all    included  .1         80   去掉#

(4)、找到# view mib2   included  .iso.org.dod.internet.mgmt.mib-2 fc  去掉#

(5)、在view  systemview  include .1.3.6.1.2.1.1下面增加一列view  systemview  included .1（有些不加也没事）

3、重启snmpd服务和验证

systemctl restart snmpd.service   //重启SNMP服务

systemctl enable snmpd.service  //开机启动SNMP服务

查看snmpd状态 systemctl status snmpd   //命令用于查看服务的当前状态和相关信息

 

再设置好snmp时可以在cati上进行验证
