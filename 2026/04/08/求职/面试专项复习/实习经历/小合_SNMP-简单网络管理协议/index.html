小合_SNMP-简单网络管理协议

|浏览量:

Simple Network Management Protocol

核心组件

  • NMS (Network Management System，网络管理系统/管理站)： 相当于“大脑”。它是运行在调度服务器上的软件，负责下发指令（Get/Set）并接收设备上报的信息。
  • Agent (代理进程)： 相当于“驻外办事员”。它是运行在交换机、路由器或服务器硬件上的软件模块，负责收集本设备的运行数据，并响应 NMS 的请求。
  • MIB (Management Information Base，管理信息库)： 相当于“字典”。它定义了设备上所有可监控项的结构和属性。
  • OID (Object Identifier，对象标识符)： 相当于“坐标/地址”。它是 MIB 树状结构中的一个具体点。例如，接口流入流量的 OID 通常是以 1.3.6.1.2.1.2.2... 开头的一串数字。

交互机制

SNMP 的逻辑非常严谨，主要通过以下几类操作完成数据交换：

📥 NMS 主动获取 (Pull 模式)

  • Get-Request： NMS 询问 Agent：“请告诉我出口 1 的流量是多少？”
  • Get-Next-Request： “请告诉我字典（MIB）中下一个条目的数据。”（常用于遍历表项）。
  • Get-Bulk-Request (v2/v3)： “一次性给我这一组所有的流量数据。”（效率更高，适合大规模调度）。

📤 Agent 主动上报 (Push 模式)

  • Trap： Agent 发现异常（如端口 Down 或流量瞬间过载）时，无需等待询问，主动发消息给 NMS：“警报！出口 1 挂了！”
  • Inform-Request： 增强版的 Trap。NMS 收到后必须回执确认，否则 Agent 会重发，确保告警不丢失。

版本演变

特性 SNMP v1 SNMP v2c SNMP v3
安全性 极低。使用明文“共同体名”（Community String）。 低。依然是明文明文共同体名，容易被窃听。 高。基于用户的安全模型（USM），支持加密和认证。
效率 低（单表项查询）。 中（支持 GetBulk 批量查询）。 中/高。
适用场景 内部实验、极其老旧的设备。 当前主流（性能好，但需靠 ACL 限制访问）。 金融、核心骨干网（对安全性要求严苛的场景）。

基本查询命令

  • snmpget：获取单个具体的 OID 值。
  • snmpwalk：遍历（步进）整个 MIB 树，常用于查看接口列表。
  • snmpbulkwalk：snmpwalk 的高效版，利用 v2c/v3 的批量获取特性，调度系统抓取大量端口流量时首选
