一阶段：HEXO初始化

Hexo 是一个基于 Node.js 的快速、简洁且高效的静态博客框架，它能将 Markdown 文档一键解析为美观的网页
以下是 HEXO 框架详细安装流程

了解目录结构

安装完成后，你的项目文件夹中会出现以下关键文件和目录

目录/文件 作用
_config.yml 站点配置文件，用于修改博客标题、作者、语言、部署地址等。
**source/ ** 资源文件夹，你的 Markdown 文章（_posts）和图片都放在这里。
themes/ 主题文件夹，存放各种博客皮肤。
public/ 静态生成目录，执行 hexo g 后生成的 HTML 文件会放在这里。
package.json 应用程序信息，记录了你安装的各种插件。

本地生成与预览

在正式发布之前，你可以在本地终端执行命令查看博客的效果

  • 清理缓存：hexo clean（建议每次重大修改后都执行，清除旧的生成文件）。
  • 生成静态页面：hexo g (全称 hexo generate)。
  • 启动本地服务器：hexo s (全称 hexo server)。
  • 访问链接：在浏览器地址栏输入 http://localhost:4000 即可看到你的博客雏形。
    此时的博客只是一个空壳，没有任何内容，我们需要创建一个远程的Git仓库用于关联博客网站以添加内容

二阶段：关联远程Git仓库

创建一个GitHub仓库

  1. 登录 GitHub，点击右上角的 + -> New repository。
  2. Repository name (仓库名)：
    • 核心关键：输入 你的用户名.github.io（例如你的用户名是 jack，就填 jack.github.io）。
    • 这样配置后，GitHub 会自动开启 Pages 服务，你的博客地址就是这个名字。
  3. Public/Private：选 Public。
  4. 初始化设置：下面的 Add a README 等选项全部不要勾选，保持一个完全的空仓库。
  5. 点击 Create repository。

本地：配置SSH密钥-建立信任

为了让你在部署时不需要频繁输入密码，且避开 ERR_CONNECTION_REFUSED 等网络权限错误，必须配置 SSH。

  1. 检查本地密钥：在 Git Bash 中输入 ls -al ~/.ssh。如果没有文件，则执行下一步。
  2. 生成密钥：
  1. 添加到 GitHub：
    • 找到本地 C:\Users\用户名\.ssh\id_rsa.pub 文件，用记事本打开并复制全部内容。
    • 在 GitHub 点击头像 -> Settings -> SSH and GPG keys -> New SSH key。
    • Title 随便起（如 my_pc），Key 粘贴进去并保存。
  2. 验证连接：输入 ssh -T git@github.com，看到 “Hi [你的用户名]! You’ve successfully authenticated…” 即表示通了。

了解双文件策略

四阶段：配置图床

配置图床是实现“一次编写，到处发布”的核心
目前最主流、最稳定的方案是：GitHub 仓库（存放图片）+ PicGo（上传工具）+ jsDelivr（CDN 加速）。

远端准备：创建一个私有/公开的 GitHub 仓库

你不需要把图片和博客代码放在一起，单独创建一个仓库更方便管理。

  1. 在 GitHub 新建仓库，名字建议叫 blog-images。
  2. 生成 Token (访问密钥)：
    • 进入 GitHub Settings -> Developer settings -> Personal access tokens -> Tokens (classic)。
    • 点击 Generate new token，勾选 repo 权限。
    • 重要：复制生成的 Token 并保存好，它只会出现这一次

工具配置：PicGo 详细设置

PicGo 是连接 Obsidian 和 GitHub 的桥梁。

基础图床设置

打开 PicGo 软件，在“图床设置”中选择 GitHub：

  • 设定仓库名：用户名/blog-images
  • 设定分支名：main
  • 设定 Token：粘贴刚才生成的密钥。
  • 指定存储路径：建议填 img/（这样图片会存放在仓库的 img 文件夹下）。
  • 设定自定义域名（关键加速步骤）： 填入：https://cdn.jsdelivr.net/gh/用户名/blog-images@main

    这样你的图片就会通过 jsDelivr 的全球节点加速，彻底解决图片加载慢的问题。

开启 Server

进入 PicGo 设置 -> 设置 Server：

  • 确保 开启 Server 为 ON。
  • 控制端口 必须为 36677。
  • 这样 Obsidian 插件才能把图片“喂”给 PicGo。
