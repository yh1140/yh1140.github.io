1、导流vlan不均衡配置

1)华为：

load-balance ecmp

ipv4 src-ip dst-ip vlan l4-src-port l4-dst-port protocol src-interface

ipv6 src-ipv6 dst-ipv6 vlan l4-src-port l4-dst-port protocol src-interface

2）H3C：

Ip load-sharing mode per-flow dest-ip src-ip dest-port src-port global

 

2、AI导流vlan不均衡配置

1）AI导流VLAN配置双IP

2）vlan的ip大小顺序从小到大（1，2，3，4）

例如：vlan711配置1，4  vlan712配置2，3

 

注意：二次导流  如果流量恒定比例 其实是好处理的，你把 4个ip 看成4股流量，如果还是错的 只能说明组合错了， hash 有时候工作方式并不是我们先的那样按顺序

两两组合不对，还是可以 1,3组合，一个vlan 3个ip，一个vlan 1个ip，当他是4股流量， 看你怎么组合而已

华为设备etrunk成员口流量不均衡，可配置eth-trunk hash-mode 7，如设备不支持此配置，可配置eth-trunk universal-id 3

load-balance profile default

 eth-trunk hash-mode 7

load-balance profile default

eth-trunk universal-id 3

五、策略应用不上

核心交换机上面配置策略时：可能出现策略应用失败

方法一：用资源共享通道方式解决（欺骗芯片）

如果方法一解决不了，可以查看一下核心配置有没有其他配置占用了很大资源；比如GRE 隧道接口; 可以把这个配置删除掉，然后在应用一下策略

remark策略应用在出口outbound资源不足：

1、配分流聚合口inbound上呢  必须先应用分流策略再应用remark

2、注意 有顺序问题 应用分流策略前 必须取消remark并commit, 再应分流策略commit, remark commit

 

交换机重启后策略未生效

检查策略是否生效（dis traffic-policy applied-record）

在业务口将未生效的策略，取消重新应用。

再次确认策略是否生效

六、交换机高内存告警

（内存在75%以下算安全）

交换机内存太高可能会导致设备正常运行，影响我们正常调度

如果有超75%的内存告警，处理方法：

（1）查看 上次重启时间(看有没有超过3个月),

命令：dis version

（2）查看有无异常告警：

命令：dis alarm history

满足，重启时间> 3个月； 没有异常告警 ，就可以重启一下交换机（reboot）
