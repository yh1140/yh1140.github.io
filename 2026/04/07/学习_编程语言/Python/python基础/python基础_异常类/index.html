python基础_异常类

发表于2026-04-07|更新于2026-04-07|编程语言Python
|浏览量:

在 Python 的逻辑体系中，异常（Exception） 是一种特殊的信号，用于中断正常的程序流并告知开发者：“逻辑执行不下去了”。

异常的继承逻辑体系

Python 的所有异常都继承自 BaseException，但作为开发者，我们通常只关注 Exception 及其子类。

BaseException: 顶级基类。

  • SystemExit: 解释器请求退出。
  • KeyboardInterrupt: 用户中断（如 Ctrl+C）。
  • Exception: 所有非系统退出类异常的基类（我们捕获的主要对象）。

常见的核心异常类

基础类型异常

异常类 逻辑含义 常见诱因
SyntaxError 语法错误 括号没闭合、冒号丢失、缩进错误。这是唯一在代码运行前就会被解析器拦截的错误。
IndentationError 缩进错误 SyntaxError 的子类，通常是空格和 Tab 混用或缩进层级不一致。
TypeError 类型不匹配 对类型不符的对象执行操作（如 '2' + 1）。
ValueError 值无效 类型正确但内容非法（如 int("abc")，字符串是合法的，但内容无法转为整数）。

资源访问异常

异常类 逻辑含义 常见诱因
NameError 未定义变量 访问了一个尚未被赋值或不在当前作用域内的变量名。
AttributeError 属性不存在 尝试访问对象没有的方法或变量（如对列表调用 .upper()）。
ImportError 导入失败 无法加载模块或名称（ModuleNotFoundError 是其子类）。
FileNotFoundError 文件不存在 试图打开一个物理路径不存在的文件。

容器与查找异常

异常类 逻辑含义 常见诱因
IndexError 索引越界 访问序列（列表/元组）时使用了超出范围的索引。
KeyError 键名不存在 在字典中访问了一个不存在的键。
