面向对象

在面向对象的世界观里，程序不再是一行行死板的指令，而是一群相互协作的对象。

类和对象

  • 类 (Class)：是一个抽象蓝图。它定义了一类事物共同擁有的属性（数据）和行为（函数）。
  • 对象（Object）：是根据蓝图创建出的具体实体。它不仅包含数据（属性），还包含处理这些数据的能力（方法）。
  • 核心逻辑：“我不关心你是怎么做的，我只关心你能做什么。” 当你调用一个对象的方法时，你是在向它发送一条“消息”，具体逻辑由对象内部处理。

四大特性

  • 封装 (Encapsulation)
    • 逻辑：将数据和操作数据的逻辑“打包”在一起，并隐藏内部细节。
    • 目的：保护数据安全。外部只能通过对象提供的“接口”（公开的方法）来访问数据，而不能直接修改内部变量。
  • 抽象 (Abstraction)
    • 逻辑：只展示对象必要的特征，隐藏无关的实现细节。
    • 示例：当你踩油门时，你不需要知道内燃机的活塞是如何运动的。你只需要知道“踩油门 = 加速”这个接口。
  • 继承 (Inheritance)
    • 逻辑：允许一个类（子类）继承另一个类（父类）的属性和方法。
    • 目的：实现代码复用，并建立逻辑上的等级制度（IS-A 关系）。
  • 多态 (Polymorphism)
    • 逻辑：同一个接口，在不同的对象下有不同的表现。
    • 示例：你对“狗”和“猫”同时发出“叫”指令。狗会“汪汪”，猫会“喵喵”。指令是一样的，但执行逻辑取决于对象的具体类型。

学习流程

①基本

  • self 的本质：理解它是指向实例内存地址的指针。
  • __init__ vs __new__：虽然 99% 的场景只用 __init__（初始化），但了解 __new__（创建实例）能帮你理解单例模式等底层逻辑。
  • 属性访问控制：掌握 _single_leading_underscore（约定私有）和 __double_leading_underscore（名称修饰）。

②方法多样性

  • @classmethod：操作类本身，常用于实现多个构造函数。
  • @staticmethod：纯粹的逻辑封装，既不访问实例也不访问类。
  • @property：实现 Getter/Setter 逻辑，这是 Python 风格的“封装”核心。

③魔法方法

Python 类的强大之处在于它能“伪装”成内置类型。你需要学习：

  • 表现类：__str__, __repr__
  • 容器类：__len__, __getitem__, __iter__
  • 运算类：__add__, __eq__
  • 生命周期：__del__, __call__

④思维

  • 组合优于继承
  • 鸭子继承
  • SOLID
  • MRO 与 super() 的线性化逻辑
