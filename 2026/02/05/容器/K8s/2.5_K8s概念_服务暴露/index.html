K8s概念_服务暴露

发表于2026-02-05|更新于2026-03-09|K8s
|浏览量:

在 K8s 的运维体系中，服务暴露（Service Exposure） 是解决“如何让外部用户访问集群内部 Pod”的核心问题。

因为 Pod 的 IP 是动态且不可靠的，所以我们需要一个持久的入口。

服务暴露的底层原理

K8s 主要通过 Service 和 Ingress 两个对象实现暴露，其核心底层技术是 kube-proxy 和 iptables/IPVS。

方式 原理 适用场景
ClusterIP 默认模式。仅集群内部访问。 数据库、内部中间件。
NodePort 在每个 Node 上开启一个相同端口（30000-32767）。 简单测试、小规模服务。
LoadBalancer 配合云厂商 API 自动分配外网 IP。 生产环境、公有云部署。
Ingress 七层转发（HTTP/HTTPS），基于域名和路径。 复杂 Web 应用、多业务共享一个 IP。
