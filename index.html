①运维面试必问：网站突然变慢你要怎么排查“网站突然变慢”可能很多人一上来就说一些排查的命令：top/netstat虽然没错，但一下子就能被面试官看出来你没有实际经验。 🧨正确的方式：必须有排查思路我会先判断：慢在哪里 ① 是所有用户慢？② 还是只有某个接口慢？③ 什么时候开始的，是刚刚变慢，还是一直慢？ 👉 👉 必须先找到最可能的故障方向 接下来再这样排查：按调用链路去层层分析用户 → CDN → 负载 → 应用 → 数据库 👉 👉重点是要一层一层地排查 🧨实战思路情况1：全部都慢我第一反应：服务器扛不住了这时候直接看 CPU / 内存（很多时候就是 CPU 100%） 情况2：某个接口慢直接怀疑数据库有没有慢查询SQL 有没有走索引 情况3：只有部分地区慢基本就是网络/CDN问题 情况4：突然变慢必问两件事：刚刚有没有上线？流量是不是暴涨了？ 🧨面试重点不是你会多少命令👉 而是你有没有排查思路：✔ 会分情况✔ 会缩小范围✔ 有经验进行判断 下面再分享一些故障例子：一个 SQL 没有走索引 → 数据库拉爆CPU 被打满 → 服务崩溃...
